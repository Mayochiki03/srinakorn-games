# ศรีนครเกมส์ ครั้งที่ 30 — เว็บอัปเดตผลการแข่งขันสด

เว็บไซต์แสดงผล + อัปเดตผลการแข่งขันกีฬาสี ของ **โรงเรียนศรีนครมูลนิธิ**
ธีมสี ม่วง–เหลือง ประจำโรงเรียน

**ข้อมูลเก็บบน MongoDB Atlas (คลาวด์)** ไม่ใช่ไฟล์ในเซิร์ฟเวอร์อีกต่อไป — เพื่อให้ข้อมูล
ไม่หายเวลาโฮสต์ฟรี (เช่น Render) รีสตาร์ทหรือหลับตัวเองหลังไม่มีคนใช้งาน

## โครงสร้างไฟล์

```
srinakhon-games/
├── server.js         เซิร์ฟเวอร์ (Node.js + MongoDB driver)
├── seed-data.js        ข้อมูลตั้งต้นจากตารางแข่งขัน (ใช้ครั้งเดียวตอนฐานข้อมูลว่างเปล่า)
├── backup.js            สคริปต์สำรองข้อมูลจาก MongoDB มาเป็นไฟล์ในเครื่อง (ไม่บังคับใช้)
└── public/
    ├── index.html        หน้าแสดงผลหลัก (ทุกคนดูได้)
    ├── admin.html         หน้าแอดมินสำหรับอัปเดตคะแนน (ต้องใส่ PIN)
    ├── css/, js/
```

## ขั้นตอนที่ 1: สมัคร MongoDB Atlas (ฟรี ไม่ต้องผูกบัตร)

1. ไปที่ [mongodb.com/cloud/atlas/register](https://www.mongodb.com/cloud/atlas/register)
   สมัครด้วยอีเมลหรือ Google/GitHub
2. เลือกสร้างคลัสเตอร์แบบ **M0 (Free)** — ฟรีตลอดไป ไม่มีวันหมดอายุ ไม่หลับ
3. ตั้งชื่อผู้ใช้ฐานข้อมูล + รหัสผ่าน (จดไว้ จะใช้ในขั้นตอนถัดไป)
4. ที่หน้า **Network Access** ให้เพิ่ม IP `0.0.0.0/0` (Allow access from anywhere)
   — จำเป็นเพราะ Render ไม่มี IP คงที่ตายตัว
5. ที่หน้า **Database** กด **Connect → Drivers** เลือก Node.js จะได้ connection string
   หน้าตาประมาณนี้:
   ```
   mongodb+srv://<username>:<password>@cluster0.xxxxx.mongodb.net/?retryWrites=true&w=majority
   ```
   แทน `<username>` และ `<password>` ด้วยค่าที่ตั้งไว้ในข้อ 3

## ขั้นตอนที่ 2: รันในเครื่องตัวเอง (ทดสอบก่อน deploy)

```
npm install
MONGODB_URI="mongodb+srv://user:pass@cluster0.xxxxx.mongodb.net/" node server.js
```

รอบแรกที่รัน ระบบจะเห็นว่าฐานข้อมูลว่างเปล่า แล้วจะนำเข้าโปรแกรมแข่งขันทั้งหมด
(64 รายการจากเอกสาร) ให้อัตโนมัติแค่ครั้งเดียว หลังจากนั้นฐานข้อมูลจะเป็นแหล่งข้อมูลจริง
แก้ `seed-data.js` อีกจะไม่มีผลอะไรแล้ว (เพราะ import ไปแล้ว)

เปิดเบราว์เซอร์ที่ `http://localhost:3000`

## ขั้นตอนที่ 3: Deploy ขึ้น Render

1. ที่หน้า Render dashboard ของเซอร์วิสเดิม ไปที่ **Environment**
2. เพิ่มตัวแปรแวดล้อม `MONGODB_URI` แล้ววาง connection string จากขั้นตอนที่ 1
3. (ไม่บังคับ) เพิ่ม `ADMIN_PIN` ถ้าต้องการเปลี่ยนรหัส PIN จากค่าเริ่มต้น `2569`
4. กด **Manual Deploy → Deploy latest commit** (หรือ push โค้ดใหม่ขึ้น git ให้ deploy อัตโนมัติ)
5. ต่อจากนี้ ต่อให้ Render สปินดาวน์/รีสตาร์ทกี่ครั้ง **ข้อมูลจะไม่หาย** เพราะอยู่บน MongoDB Atlas
   ไม่ใช่ในตัวเซิร์ฟเวอร์อีกต่อไป

> Render free tier ยังคง "หลับ" หลังไม่มีคนเข้าใช้ 15 นาที (โหลดครั้งแรกหลังตื่นจะช้าไปนิดนึง
> ~30 วินาที) แต่**ข้อมูลจะปลอดภัยเสมอ** ไม่ว่าจะหลับกี่รอบ เพราะไม่ได้เก็บในตัวเซิร์ฟเวอร์แล้ว

## สำรองข้อมูล (แนะนำ ก่อนวันงานจริง)

MongoDB Atlas free tier (M0) **ไม่มีระบบสำรองข้อมูลอัตโนมัติ** ต่างจากแพลนเสียเงิน
แนะนำรันคำสั่งนี้เป็นระยะ (เช่น วันละครั้งช่วงมีการแข่งขัน) เพื่อดาวน์โหลดสำเนาไว้ในเครื่อง:

```
MONGODB_URI="mongodb+srv://user:pass@cluster0.xxxxx.mongodb.net/" node backup.js
```

จะได้ไฟล์ใน `data/backups/backup-<วันเวลา>.json` — เป็น read-only ไม่กระทบข้อมูลจริง

## รหัส PIN สำหรับแอดมิน

ค่าเริ่มต้นคือ `2569` เปลี่ยนได้ด้วยตัวแปรแวดล้อม `ADMIN_PIN`
(ตั้งใน Render → Environment หรือตอนรันในเครื่อง)

## การอัปเดตผลการแข่งขัน

1. เข้า `/admin.html` แล้วใส่รหัส PIN
2. ค้นหา/กรองรายการที่ต้องการด้วยชิพเลือกชนิดกีฬา หรือช่องค้นหา
3. กรอกคะแนนแต่ละสี แล้วกด **บันทึก** — ระบบจะคำนวณผู้ชนะให้อัตโนมัติ
   (แต่ **ยังไม่นับเข้าตารางคะแนนสะสม** จนกว่าจะยืนยันว่าจบแล้ว ดูข้อ 4)
4. **สำคัญ**: เมื่อแข่งจบจริง ให้เลือกที่ช่องสถานะเป็น **"✅ จบแล้ว (ประกาศผล)"** แล้วกดบันทึก
   ผลจะถือเป็นทางการ และตอนนั้นจะถูกนับเข้าตาราง "คะแนนสะสมประจำสี"
5. ถ้าอยากกำหนดผู้ชนะเองโดยไม่อิงจากคะแนน ใช้ช่อง "ผู้ชนะ" เลือกทีมที่ชนะได้โดยตรง

หน้าแสดงผลหลัก (`index.html`) ดึงข้อมูลใหม่ทุก 20 วินาทีอัตโนมัติ พร้อมคอนเฟตตี + แบนเนอร์
แจ้งเตือนเมื่อมีการยืนยันผลใหม่

## เพิ่ม / แก้ไข / ลบ รายการแข่งขัน

กด **"＋ เพิ่มรายการแข่งขัน"** ที่หน้าแอดมิน จะเปิดฟอร์มให้กรอกได้ครบ:
ชนิดกีฬา, หมวด/รุ่น (เช่น ม.ต้น, ม.ปลาย), วันที่, เวลา, สถานที่, ชื่อผู้ควบคุม, คู่ที่/รอบ,
หมายเหตุ และเลือกได้ว่าเป็น "การแข่งขัน" (มีคู่แข่ง 2 สี + คะแนน) หรือ "กิจกรรม" (ไม่มีคู่แข่ง)

ทุกรายการมีปุ่ม **✎ แก้ไข** และ **🗑 ลบ** ต่อท้ายเสมอ กด **"⚙ ชนิดกีฬา"** เพื่อเพิ่ม/แก้/ลบ
ชนิดกีฬาใหม่ (เช่นแยก "E-SPORT ม.ต้น" กับ "ม.ปลาย" เป็นคนละรายการ)

## กีฬาพื้นบ้านแบบแข่งพร้อมกันหลายสี (FFA)

บางรายการ (เช่น ตีกอล์ฟคนจน, กีฬาพื้นบ้าน) ไม่ได้แข่งเป็นคู่ 2 สี แต่แข่งพร้อมกันทีเดียว
2-4 สี แล้วมีผู้ชนะแค่สีเดียว ให้สร้างรายการแบบนี้โดยกดปุ่ม
**"🎯 กีฬาพื้นบ้าน (แข่งพร้อมกันหลายสี)"** ตอนเพิ่ม/แก้ไขรายการ แล้ว:

1. ตั้งชื่อกิจกรรม/เกม เช่น "ตีกอล์ฟคนจน"
2. เลือกสีที่เข้าแข่งขัน (เลือกได้ตั้งแต่ 2 สีขึ้นไป ไม่ต้องครบ 4 สีก็ได้)
3. กำหนดคะแนนที่สีผู้ชนะจะได้ (ค่าเริ่มต้น 3 คะแนน)

ในหน้าแอดมิน กรอกคะแนน/ผลของแต่ละสีแล้วกดบันทึก ระบบจะคำนวณผู้ชนะให้อัตโนมัติจาก
คะแนนสูงสุด (หรือกำหนดผู้ชนะเอง/เสมอ ได้ที่ช่อง "ผู้ชนะ") เหมือนรายการแข่งขันแบบคู่ปกติ
ผลจะยังไม่นับเข้าตารางคะแนนสะสมจนกว่าจะเลือกสถานะ **"✅ จบแล้ว (ประกาศผล)"** เช่นกัน
เมื่อยืนยันแล้ว คะแนนของสีผู้ชนะจะถูกบวกเข้าตาราง "คะแนนสะสมประจำสี" ทันที

## แชมป์ประจำชนิดกีฬา (ประกาศเอง ไม่คำนวณอัตโนมัติ)

กติกาตัดสินแชมป์ของแต่ละชนิดกีฬาต่างกัน (ผลต่างประตู, จุดโทษ, ผลการพบกัน ฯลฯ)
ระบบจะ**ไม่เดา**ว่าใครเป็นแชมป์อัตโนมัติ ให้ครูกดยืนยันเองที่ **"⚙ ชนิดกีฬา"**
เลือกสีที่ได้แชมป์ + กำหนดคะแนนที่จะได้ (ค่าเริ่มต้น 5 คะแนน)

## สถิติผู้เข้าชมเว็บ

- ตัวเลข **"ผู้เข้าชมวันนี้"** ขึ้นในกล่องวันที่ด้านบนหน้าเว็บ (นับไม่ซ้ำต่อวัน)
- หน้าแอดมินกด **"📊 สถิติผู้เข้าชม"** ดูยอดวันนี้/ทั้งหมด + กราฟย้อนหลัง 14 วัน
- ใช้รหัสสุ่มในเบราว์เซอร์ผู้เข้าชม ไม่ผูกบัญชี ไม่เก็บข้อมูลส่วนตัว
